利用反向代理将启用 HTTPS 的域名指向到 GitHub Page 的二级域名

Pissy ca-ca
2017-10-09
反向代理, 技术

今天将 lxwu.me 启用了 HTTPS 访问，由于签发的证书需要独立的 VPS 存储（我用的 Linode ），所以要将域名解析添加一条 A 记录，记录值为 VPS 的 ip 地址，最后再通过反向代理（我用的 Nginx ）指向到 GitHub Page 的二级域名。

网站如何启用 HTTPS 访问可以参考《如何免费的让网站启用HTTPS》
Nginx 反向代理配置可以参考我的配置文件

需要注意的是由于 GitHub 限制，proxy_set_header Host的值不是平常配置反向代理的$host，而是 GitHub Page 的二级域名。
